Я новичок в веб программировании и laravel изучаю только максимум неделю, так что не судите строго, если есть какие-то ошибки в оформлении вопроса или просто в коде укажите на них, пожалуйста. Теперь сама суть, у меня есть ресурс контроллер NewsController, в нем все дефолтные методы и пару своих, суть в том что методы edit и show не работают, вот их листинг

    public function show(News $news)
    {
        dd(__METHOD__);
    }
    public function edit(News $news)
    {
        dd(__METHOD__);
    }
News это модель статей,в ней переопределен метод

    public function getRouteKeyName()
    {
        return 'slug';
    }
То есть в методы edit и show я должен автоматом получать модель по их slug.

Вот мои роуты

    Route::group(['namespace' => 'Admin', 'prefix' => 'admin'], function(){
        Route::group(['namespace' => 'News', 'prefix' => 'news'], function () {
            Route::resource('/', 'NewsController', [
                'only' => [
                    'index',
                    'create',
                    'store',
                    'show',
    
                ],
                'names' => [
                    'index' => 'news.table',
                    'create' => 'news.create',
                    'store' => 'news.save',
                    'show' => 'news.change',
    
                ]
            ]);
    });
Вот эти же роуты при помощи route:list
```
    | GET|HEAD  | admin                            | home                  | App\Http\Controllers\Admin\AdminController@index                       | web,auth,admin
    | GET|HEAD | admin/news                       | news.table            | App\Http\Controllers\Admin\News\NewsController@index                   | web,auth,admin
    | POST     | admin/news                       | news.save             | App\Http\Controllers\Admin\News\NewsController@store                   | web,auth,admin
    | POST     | admin/news/ckeditor/image_upload | upload                | App\Http\Controllers\Admin\News\NewsController@upload                  | web,auth,admin
    | GET|HEAD | admin/news/create                | news.create           | App\Http\Controllers\Admin\News\NewsController@create                  | web,auth,admin
    | GET|HEAD | admin/news/multiple-destroy      | news.multiple.destroy | App\Http\Controllers\Admin\News\NewsController@multipleDestroy         | web,auth,admin
    | GET|HEAD | admin/news/{}                    | news.change           | App\Http\Controllers\Admin\News\NewsController@show                    | web,auth,admin
```
У роута news.change, который и есть show нет даже параметра.
В похожем вопросе ( https://ru.stackoverflow.com/questions/454500/laravel-5-проблема-с-контроллером-ресурсов ) было ответом убрать "/" из роутов, в моем случае это ничего не меняет. Но если я не буду использовать "prefix" в роутах, то есть 
```
    Route::group(['namespace' => 'Admin'], function(){
        Route::group(['namespace' => 'News'], function () {
            Route::resource('admin/news', 'NewsController', [
                'only' => [...],
                'names' => [...]
            ]);
    });
```
Роуты отображаются верно
```
|        | GET|HEAD   | admin                          | home                  | App\Http\Controllers\Admin\AdminController@index                       | web,auth,admin
|        | GET|HEAD   | admin/news                     | news.table            | App\Http\Controllers\Admin\News\NewsController@index                   | web,auth,admin
|        | POST       | admin/news                     | news.save             | App\Http\Controllers\Admin\News\NewsController@store                   | web,auth,admin
|        | GET|HEAD   | admin/news/create              | news.create           | App\Http\Controllers\Admin\News\NewsController@create                  | web,auth,admin
|        | GET|HEAD   | admin/news/{news}              | news.change           | App\Http\Controllers\Admin\News\NewsController@show                    | web,auth,admin

```
Или есть еще рабочий вариант просто вынести в отдельный роут, с использованием "prefix"
```
Route::get('{slug}', 'NewsController@show')
            ->name('news.change')
```
Всё будет работать (поверьте). Я понимаю, что можно забить и писать префиксы в ручную, но это не очень удобно и тем боле я буду иметь не разрешенную проблему.
